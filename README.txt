================================================================================
                    Lightweight CI/CD 프로젝트 문서 가이드
================================================================================

이 프로젝트는 GitHub Webhook 기반 자동 빌드/배포 시스템입니다.
Docker, Rust, WebSocket을 활용한 경량 CI/CD 솔루션입니다.

================================================================================
문서 구조
================================================================================

1. 최종_설계_문서.txt
   - 프로젝트 전체 개요 및 아키텍처
   - 시스템 구성 요소 설명
   - 개발 단계별 계획
   - **먼저 읽어야 할 문서**

2. API_통신_명세서.txt
   - 모든 HTTP REST API 엔드포인트
   - WebSocket 프로토콜 상세
   - GitHub Webhook 처리
   - Docker API 호출 명세
   - 데이터베이스 스키마
   - 내부 이벤트 시스템
   - **개발 시 참조할 기술 명세서**

================================================================================
핵심 개념
================================================================================

아키텍처:
  - DOOD (Docker-outside-of-Docker)
  - Blue/Green 무중단 배포
  - WebSocket 실시간 통신
  - 프로젝트별 경로 기반 라우팅

도메인:
  - ci.yourdomain.com → 관리 대시보드 (포트 3000)
  - app.yourdomain.com → 실제 서비스 (포트 8080)

기술 스택:
  - Rust: axum, tokio, bollard, sqlx
  - Docker: 빌드 및 런타임 환경
  - SQLite: 메타데이터 저장
  - WebSocket: 실시간 로그/상태 스트리밍

================================================================================
시작 가이드
================================================================================

1. 설계 이해하기
   → 최종_설계_문서.txt 읽기

2. API 명세 확인
   → API_통신_명세서.txt 읽기

3. 개발 시작
   → API 명세서를 기준으로 각 모듈 구현

================================================================================
